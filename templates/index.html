Method & honest reads

What was built, what worked, and where neural networks did not beat boosted trees on this dataset.

The deployed model

Calibrated, threshold-tuned, fairness-audited

Isotonic-calibrated XGBoost classifier tuned with Optuna; threshold selected on validation F1. Feature pipeline materialises 29 fields per encounter — numeric, binary, low-card categorical and high-card categorical (admit-dx code, DRG).

  • Cohort: CMS DE-SynPUF (synthetic Medicare claims)
  • Prevalence ≈ 10% · patient-grouped splits
  • SHAP explanations · LIME local explanations
  • Evidently drift report shipped as monitoring baseline

What V2 changed

Honest negative results, kept in plain sight

NNs do not beat GBMs here. Top NN/AutoML PR-AUC was 0.157 (MLP) vs deployed XGBoost 0.172. Tabular GBMs dominate this regime — ~100 engineered features, ~10% prevalence.

Ensembles do not lift either. Voting + stacking added +0.00007 PR-AUC over the best base. The three base learners correlate at ρ ≈ 0.99 — nothing for a meta-learner to learn.

Both findings are reported transparently rather than buried.

This dashboard

No mocks — every artefact is the real one

Every CSV under reports/ is rendered as-is. Every score uses the production Predictor. The drift iframe is the same HTML a CI pipeline would diff against. The form schema mirrors the model's FEATURE_ORDER exactly.

Pure Flask + Jinja + vanilla JS. No framework, no build step. Deployable to Render, HF Spaces, or any Python host via gunicorn flask_app:app.
